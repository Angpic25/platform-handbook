# Glossary

| Term | Definition |
|------|------------|
| **Account** | The top level container representing a client organization. One Account per company. |
| **Touchpoint** | A website or digital application subject to accessibility analysis. |
| **Compliance level** | Percentage of alignment with WCAG standards. An improvement indicator, not a legal certification. |
| **Open issue** | An accessibility issue that has been identified but not yet addressed. |
| **Fixed issue** | An issue corrected by the client dev team but not yet validated by an expert. |
| **Resolved issue** | An issue that has been corrected and expert validated through a retest. |
| **Dismissed issue** | An issue that has been archived or ignored (no action required). |
| **Recurring issue** | An issue that appears on multiple pages of the touchpoint. |
| **Unique issue** | An issue specific to a single page. |
| **Mitigation** | A temporary fix applied by the AccessiWay widget while the team implements a permanent code level fix. |
| **Retest** | An expert review to confirm that fixes have been correctly implemented (max 1 per touchpoint). |
| **WCAG** | Web Content Accessibility Guidelines, the international accessibility standard (version 2.2 in the platform). |
| **EN 301 549** | European standard for accessibility requirements for ICT products and services. |
| **User flow** | A typical user navigation path within the website (e.g. "About Us", "Internet Banking"). |
| **Critical severity** | Issues that block essential user tasks and must be fixed urgently. |
| **High severity** | Issues that affect main navigation and should be prioritized. |
| **Low severity** | Minor usability issues that can be addressed at a later stage. |
| **2FA** | Two Factor Authentication for secure platform login. |
| **User impact** | A classification of how severely an issue affects real users of the touchpoint. |
