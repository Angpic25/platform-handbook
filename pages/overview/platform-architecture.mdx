# Platform Architecture

The platform is built around a three-level hierarchy: **Organization → Touchpoint → Analysis**.

![Diagram showing the platform hierarchy: Organization at the top, Account with members below, then Touchpoints, each with their own Analysis containing issues by severity](/images/diagram-architecture.png)

## Organization

At the top level sits the Organization, which represents the client company (for example, Westpac Bank). The Organization is the entry point that defines the scope of everything inside the platform.

## Account

Each Organization has one Account. The Account is the shared workspace that all team members access. It holds all the organization's accessibility data, settings, and integrations in one place.

Each Account has a single **Owner**, who is responsible for managing access. The Owner can invite an unlimited number of team members, each with their own login. All members share the same view of the data within the Account.

## Touchpoints

Within the Account, there can be multiple Touchpoints. Each Touchpoint represents a single website or digital application that has been audited for accessibility. For example, a banking group might have one Touchpoint for their consumer site, one for their payments portal, and one for their wealth management platform.

Each Touchpoint is managed independently, with its own compliance dashboard, trend charts, and integration settings.

## Analysis

Each Touchpoint contains an Analysis, which is where all the accessibility work happens. The Analysis includes the full list of issues identified by AccessiWay experts during the audit, grouped by severity (Critical, High, Low). Each issue comes with a description, the relevant WCAG criterion, a suggested fix, and expert comments.

The Analysis is updated every time a new audit is completed or a retest is performed.
